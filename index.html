selected project

A job-search agent that runs itself

A personal distributed system that does my job search for me. Every morning it crawls four public-sector job boards (USAJobs, CalCareers, the CSU system, and EdJoin), normalizes a few hundred raw postings, and scores each one against a structured inventory of my actual experience. Only near-perfect matches surface; the rare standout triggers an email.

The architecture splits the heavy work from the serving. A launchd cron job on my Mac runs the full crawl, filter, and score pipeline locally each day, renders a static HTML report, and publishes just that artifact to the server. The server's only job is to serve the page. Nothing scrapes in production, so there is no long-running crawler to babysit and no production cost to the model calls.

Scoring runs in two tiers. A deterministic pass (keywords, title rules, location, salary floor) filters and ranks cheaply; only the survivors reach the LLM tier, which keeps token spend near zero. Crawlers are rate-limited and robots-aware, and an empty scrape never overwrites the last good run.

  • Pipeline · daily launchd cron on Mac, publish to server, serve static
  • Scoring · deterministic filter, then a Claude LLM tier on the survivors
  • Reliability · rate-limited crawlers, robots-aware, empty-scrape protection
